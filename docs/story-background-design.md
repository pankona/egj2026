## 背景

これは 5 度目の改訂。経緯:

1. **初稿 (ファンタジー寄り)**: 説明テキスト前提のため却下
2. **2 稿 (現実寄り)**: 配電/ネットワーク。「囲って消す」が直感と合わず却下
3. **3 稿 (封印もの)**: 「方陣を引いて悪いものを封じる」がコアと判明。**ここから世界モデルは確定**
4. **4 稿 (タイトル再考、1 単語)**: `SEAL` を推奨。ただし動詞列の延長すぎてタイトル感が弱い
5. **本稿 (2 単語)**: ジャム作品らしい文学的な手触りを出すために 2 単語タイトルを設計

確定している世界モデル (3 稿から維持):

> 悪いものが世に染み出している。プレイヤーは光の線で方陣を引き、囲い込んで封じる。封じきれなければ陣が破れる。悪いもの同士も同じ儀礼を逆向きに学んで、こちらを囲い返してくる。十の場所を巡り、最後の根を封じれば終わる。

## 方針

2 単語タイトルの設計条件:

- **声に出して言ったときのリズム** — 2 単語タイトルは弱強・強弱のアクセントが効く。`Last Ward` / `Night Seal` のように 1 音節 + 1 音節が締まる
- **片方の単語に動詞性、もう片方に世界観を持たせる** — `Seal the Dark` 型 (動詞 + 目的語、3 単語まで許容) か、`Last Ward` 型 (形容詞 + 名詞)
- **既存動詞列 `DRAG. SLASH. ENCLOSE. SEAL.` と語彙的に親和** — タイトル単語が動詞列のどこかと響き合うとリズムが揃う
- **`bind` 禁止** — コード内で技術用語 (`bindPhase`)。タイトルに使うと混同
- **検索性** — 既存ゲームと完全一致しない組み合わせを選ぶ

## タイトル候補 (2-3 単語)

### 候補 1. LAST WARD 【推奨】

- **意味**: 最後の結界 / 最後の守り。`ward` は結界・印・守護を兼ねる
- **強み**:
  - 「最後の」が儀礼の終焉感を作る。10 ステージ目のボス戦が「最後の結界を打つ」に直接接続
  - 2 音節 + 1 音節、声の収まりがいい
  - 物語の輪郭を 2 単語で語っている: 主人公は「最後の (結界師)」、その「結界」を打って回る
  - `SEAL` を動詞として残せる: タイトル `LAST WARD` / 動詞列 `... SEAL.` / フラッシュ `SEALED!` の三層構造になる (タイトル = 役職、動詞列 = 所作、フラッシュ = 結果)
- **弱み**:
  - `ward` の二次的意味 (病院の病棟) がやや雑念
  - 「最後の」が終末感を強く出すので、トーンが重め (ゲーム本編は割と軽快なリズムゲーなので、ややミスマッチに感じる人もいるかも)

### 候補 2. SEAL THE DARK (3 単語)

- **意味**: 闇を封じよ。動詞 + 目的語
- **強み**:
  - **タイトル自体が指示文**。プレイヤーは「何をするゲームか」を題名から即理解できる
  - 動詞列 `... ENCLOSE. SEAL.` の最後の動詞をタイトルへ持ち上げる構造で、ゲーム全体のテーマがそのまま掲げられる
  - 大文字で並べたときに 4-3-4 の音節バランスが綺麗
- **弱み**:
  - 3 単語、若干長い (`HOLD THE LINE` と同じ長さ)
  - 命令形タイトルは Bumi の `SEAL THE GAP` など先例多数。差別化はやや弱い

### 候補 3. HOLD THE LINE (3 単語)

- **意味**: 戦線を保て / (引いた) 線を維持せよ
- **強み**:
  - 「線を引いて闇を抑える」プレイ動作と完全に一致 (line = 引いた光線、hold = 敵に破られないように維持)
  - 軍事英語の慣用句。プレイヤーは初見で文意を取れる
  - 動詞列 + タイトル両方が「線」を語る統一感
- **弱み**:
  - 慣用句なので「ありふれた」印象が出る
  - 「結界 / 封印」のファンタジー味は薄まり、サバイバル / TD ゲーム的なトーンに寄る

### 候補 4. NIGHT SEAL

- **意味**: 夜の封印 / 夜を封じる
- **強み**:
  - 闇 vs 光の対比をタイトルに直接書ける
  - 1 音節 + 1 音節、極めて短い
- **弱み**:
  - `Night Seal` は同名作品が複数あり (検索性が落ちる)
  - やや陳腐に聞こえる可能性

### 候補 5. INK AND SEAL

- **意味**: 印と封 / 墨と封蝋
- **強み**:
  - 「線を引く (ink)」と「封じる (seal)」が両方タイトルに入る、動作の完全な要約
  - 文具的な抽象性。ジャム作品らしい余韻
- **弱み**:
  - `AND` が間に入って 3 単語、やや散漫
  - `INK` は「闇」のメタファーとも読めてしまい、誰が引くものか曖昧

### 候補 6. LAST SIGIL

- **意味**: 最後の印
- **強み**: 候補 1 と同型 (最後の◯◯)。`sigil` は方陣・印章の専門語で神秘性が出る
- **弱み**: `sigil` は読みにくい英語話者がいる (`signal` と混同される)。日本語話者にも馴染み薄

### 候補 7. LIGHT WARD

- **意味**: 光の結界
- **強み**: 光と結界の二語、ミニマル
- **弱み**: `Light Ward` は形容詞 + 名詞だが、`Light` が動詞 (灯す) にも形容詞 (軽い) にも読めてしまい、意味が一意に定まらない

### 候補 8. AFTER LIGHT (または AFTERLIGHT)

- **意味**: 光の後 / 残光
- **強み**:
  - 既存コード内の `SlashGlowFrames` (= 残光フレーム) を直接タイトル化できる。ゲームの最も視覚的に印象的な現象 (斬撃の残光) と一致
  - 文学的、ジャム作品らしい余韻
- **弱み**:
  - 「封印もの」の比喩からはやや離れる (残光が主役感を出すと、儀礼感が薄まる)
  - `Afterlight` という 1 単語の写真アプリが存在する (`AFTER LIGHT` と分かち書きすれば回避)

### 検討して捨てた候補

- **DARK BIND / BINDING** — `bind` がコード内技術用語。NG
- **HOLY SEAL / DIVINE SEAL** — 宗教色が強すぎる、ゲームジャムの一般受けで滑りやすい
- **WARD WALKER** — 結界を歩く者 (主人公の二つ名)。やや凝りすぎ
- **THE LAST WARD** (冠詞付き) — 候補 1 の冠詞付き版。シンプルに `LAST WARD` の方が締まる
- **NIGHTFALL / NIGHTBOUND** — 1 単語に戻ってしまう

## 推奨案

**候補 1「LAST WARD」** を推す。

決め手:

- **2 単語タイトルとしての座りが最良**。形容詞 + 名詞、2 音節 + 1 音節、ロゴ的にも声の上でも収まる
- **物語の輪郭を 2 語で語っている**:「最後の (儀礼継承者が打つ) 結界」。ステージ 10 = 「文字通り、これが最後の結界」という構造が自動的に成立する
- **動詞列・フラッシュとの三層構造**:
  - タイトル: `LAST WARD` (役職・状況)
  - 動詞列: `DRAG. SLASH. ENCLOSE. SEAL.` (所作の宣言)
  - フラッシュ: `SEALED!` (結果のフィードバック)

  これは「いつ何を見せるか」が綺麗に分かれていて、増やすほど嫌味になる文学的修辞を、それぞれの場所に必要な量だけ配置する設計になる

- **DISCONNECT (ジャムテーマ) との接続**: 「最後の結界 = もうこれが切れたら 接続 が戻らない」と読める。テーマの儀式譚的解釈とハマる

### スライダー (どこまで採用するか)

1. **最小**: タイトル文字列 (ウィンドウ + 画面大見出し) を `RIFT` → `LAST WARD` に置換するだけ
2. **中** (推奨): 上記 + 状態遷移文言を差し替え (`CLEAR` → `SEALED.`、`TIME UP` → `WARD BROKE.`、`ALL CLEAR` → `ALL SEALED.`)
3. **全部**: 中 + `Next: Stage N` → `Next: N / 10` + README 冒頭フレーバー 1 行

中を推奨。`WARD BROKE.` と `ALL SEALED.` までやって、タイトルと状態遷移が同じ辞書から引かれているのが体感できる状態にしたい。
